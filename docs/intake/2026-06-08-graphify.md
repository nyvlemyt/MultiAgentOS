# Intake — Graphify (graphifyy) (2026-06-08)

- **Type** : tool (knowledge graph de codebase)
- **Source** : repo `graphifyy` (~35k⭐) + Google Doc @le_gouverneur_ia
- **Récence / obsolescence** : actif 2026 / **low**
- **Résumé** :
  - Transforme un dossier projet en **knowledge graph** interrogeable. Pipeline 3 passes : **AST tree-sitter** (déterministe, 25 langages) → **multimodal** (Whisper audio/vidéo, PDF/images) → **Claude sub-agents** (extraction concepts + clustering Leiden).
  - Sorties `graphify-out/` : graph HTML, rapport markdown (nœuds + communautés), JSON queryable, cache incrémental.
  - Métriques annoncées : **-71.5× tokens/requête**, compréhension codebase ~3 semaines → ~4 min. `graphify watch`, `merge-graphs`.

## Fit
- Surface : **Context Manager** sur **projets externes** (ex : comprendre OtakuGO_UP) — c'est l'implémentation "graph" du context-pack (`data/context-packs/<projectId>.md`).
- **PAS de la mémoire projet** (`data/memory/`). Ne remplace ni les 5 registres, ni QMD. Couche distincte (codebase, pas docs/mémoire Markdown).
- **Blast radius** (impact refactoring) → utile au Reviewer pour estimer l'impact d'un diff.

## Coûts
- **Install** : `uv tool install graphifyy` + tree-sitter/Whisper. Effort moyen.
- **Maintenance** : re-index par projet, `.graphifyignore`.
- **Retrait** : facile (outil séparé, sorties dans `graphify-out/`), MAIS il **spawn des Claude sub-agents** → consomme du **quota Agent SDK** à chaque index (billing §11).
