# U5 — §13 Self-audit: wave 0d vs ADR 0007

- **Date**: 2026-06-25
- **Scope**: 0d implementation (branch `phase/9d-arsenal`, base `phase/9c-roster`) audited against the six decisions of `docs/decisions/0007-arsenal-exploitation.md`. Audit-only — no product code touched. Diff inspected via `git diff phase/9c-roster..phase/9d-arsenal` (21 product/test/CI/doc files, +1552/-7).
- **Auditor**: self-audit pass (CLAUDE.md §13). Independent of the producer's Checker verdicts (`U4`/`U4a`/`U4b`) — those are cross-referenced, not trusted blindly.

## Verdict

**0d honored ADR 0007 — yes, with gaps.** The value core (Decisions 1, 2, 3) is implemented exactly as specified: `selectLibrarySkills` takes an injected `retriever?`, RRF-fuses a deterministic tag-score order (source a) with a semantic order (source b), and degrades silently-but-never-crashing to source (a) when QMD is absent/FTS/throwing. The suggestion-only boundary (Decision 4) holds with no §5 leak — cold-agent scoring is data-only, emitting a `cold_agent_suggested` event without ever mutating routing state. The golden-set floor (Decision 6) is added. Two decisions are only **partially** realized: **Decision 5** (MCP "service consommé") is wired in `llm.real.ts` but **never activatable from the dispatch path** — no caller can set `mcp: true`, so no live mission can make an agent call `query`; the ADR's "preuve attendue" is met only by a unit test on the options object, not by a runtime path. And the **semantic-recall promise is structurally unverified in CI** (the arsenal-eval job is FTS-pinned, so every `qmdOnly` row auto-skips) — recall is asserted only by a local, manual `MAS_RETRIEVAL_BACKEND=qmd` run, which depends on operator discipline. Two smaller drifts: cold-agent suggestion uses the **static tag/role scorer, not the semantic retriever** (ADR-permitted but it means "the arsenal acts by semantic relevance" is realized for skills only, not agents), and the golden score-floor guards exactly **one** of the six arsenal rows.

## Cross-check against producer verdicts

The Checker's `U4`/`U4a`/`U4b` PASS verdicts are accurate on what they assert (degraded invariant held, byte-identical no-retriever path, throwing-retriever fallback, §5 data-only with no `TIER_B_DELEGATION_MAP` write, opt-in default-OFF byte-identical, minScore floor, qmdOnly honesty). **They did not surface F1**: U4 §"4c contract review" states the MCP wiring is correct and least-privilege (true at the `claudeCodeLLM` layer) but did not trace whether any *dispatch caller* can reach it — and none can. So 4c passes its unit contract while being runtime-inert. F2 is acknowledged by the producer in the ci.yml comment as an explicit honest-skip decision, but its consequence (ADR §Décision-6's CI guarantee is partial) is not flagged as residual debt.
