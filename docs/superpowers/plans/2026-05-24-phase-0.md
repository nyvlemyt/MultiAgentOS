# Phase 0 — Foundations + Visual Vision Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:executing-plans`. Steps use `- [ ]` for tracking.

**Goal:** Walkable cockpit skeleton — 13 routes rendered with mocked data, dark+light theme, Drizzle schema + seed, worker daemon + SSE, orbit view animated. All Phase 0 exit criteria in `ROADMAP.md`.

**Architecture:** pnpm monorepo · `apps/web` (Next.js 15 App Router) + `apps/worker` (tsx daemon) · `packages/{core,db,agents,skills,memory,tokens,ui}` · SQLite via Drizzle · SSE channel for worker→web push · CSS-vars theming · mocked LLM in `packages/core/llm.ts`.

**Tech stack:** Next.js 15, React 19, TypeScript 5.5, Tailwind 4, shadcn/ui, Drizzle ORM, better-sqlite3, tsx, Vitest, Playwright (smoke), pnpm workspaces, Node 20.

**Budget:** 90 k session tokens. Pause + report at 80 % (72 k).

---

## Scope discipline (read this first)

Phase 0 establishes the visual + structural foundation **only**. Specifically OUT of Phase 0 scope:
- Real LLM calls (Phase 2)
- Interactive mission FSM transitions (Phase 1)
- Skill summary generation (Phase 3)
- Tier B delegation execution (Phase 5)

Polish is acceptable at "good and consistent", not "breathtaking". Phase 7 owns final polish.

## Risks + budget watchpoints

- **Drizzle SQLite + `better-sqlite3` on macOS** sometimes needs native rebuild on Node 20. If a rebuild loop starts, pause and report — do not retry blindly.
- **Tailwind 4** breaking changes vs 3: stick to v4 docs; if stuck, fall back to v3.4. Cost in tokens, not architecture.
- **Playwright install** downloads browsers (~120 MB, no token cost but slow). Acceptable.
- **SSE under Next.js App Router** has edge-case streaming caveats. Phase-0 fallback = DB polling from the route handler. Phase 2 upgrades to true IPC.
- **80 % budget breaker** = ~72 k tokens. If we cross it before Task 12, drop styling polish on Tasks 11–14 and ship "minimum legible" UI.
